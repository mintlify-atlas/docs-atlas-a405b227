## How Real-Time Cart Sync Works

### Intelligent Monitoring

The cart doesn't just store static data - it actively watches Firebase:

```javascript
// For each unique product in cart
productIdsInCart.forEach(productId => {
  // Create real-time listener
  cartUnsubscribers[productId] = onSnapshot(doc(db, "products", productId), (snap) => {
    // Update price, stock, and availability instantly
    updateCartItemsFromCloud(productId, snap.data());
  });
});
```

### What Gets Updated Automatically

<Tabs>
  <Tab title="Price Changes">
    **When Admin Updates Price**
    
    1. Firebase detects the change
    2. Cart receives update via onSnapshot
    3. Item price updates silently in cart
    4. Cart total recalculates automatically
    5. Customer sees new price on cart page
    
    <Note>
    Cart items added before a price increase keep their original price until the customer refreshes or leaves the page.
    </Note>
  </Tab>
  <Tab title="Stock Adjustments">
    **When Inventory Changes**
    
    1. Another customer purchases the product
    2. Stock decreases in Firebase
    3. Cart receives stock update
    4. If customer's quantity exceeds new stock:
       - Quantity is automatically adjusted down
       - Customer is notified
    5. If stock reaches zero:
       - Item is marked "Out of Stock"
       - Cannot proceed to checkout
  </Tab>
  <Tab title="Product Deactivation">
    **When Product is Removed**
    
    1. Admin deactivates or deletes product
    2. Cart detects status change
    3. Item is marked as unavailable
    4. Stock is set to 0
    5. Customer must remove it before checkout
  </Tab>
  <Tab title="Variant Changes">
    **When Specific Variant Changes**
    
    1. Admin updates a color/capacity combination
    2. Cart checks if customer's specific variant changed
    3. Updates price/stock for that exact combination
    4. Other variants in cart remain unchanged
  </Tab>
</Tabs>

### Smart Cleanup

When customer removes items from cart:
- Real-time listener for that product is immediately stopped
- Reduces Firebase connection usage
- Optimizes performance and costs

### Shipping Cost Calculation

#### How Shipping is Calculated

<Tabs>
  <Tab title="Free Shipping">
    **Order Value Threshold**
    
    ```javascript
    if (cartTotal >= shippingConfig.freeThreshold) {
      shippingCost = 0;
      // Display "FREE SHIPPING" badge
    }
    ```
    
    When cart value exceeds the threshold (configured by admin), shipping is free nationwide.
  </Tab>
  <Tab title="City-Based Pricing">
    **Grouped Rates**
    
    ```javascript
    // Example shipping groups
    groups: [
      { cities: ['Bogotá', 'Cali', 'Medellín'], price: 15000 },
      { cities: ['Barranquilla', 'Cartagena'], price: 20000 },
      // ... more groups
    ]
    ```
    
    Cities are grouped into pricing tiers. System matches selected city to appropriate group.
  </Tab>
  <Tab title="Default Rate">
    **Fallback Pricing**
    
    If city not found in any group, default shipping rate applies. Ensures all locations are covered.
  </Tab>
</Tabs>

#### Real-Time Shipping Config Sync

Shipping configuration loads from Firebase with real-time monitoring:

- **Initial Load**: From sessionStorage cache (instant)
- **Background Sync**: onSnapshot listens for admin changes
- **Automatic Update**: If admin changes shipping rates, customer sees new rates immediately
- **No Page Reload**: Updates happen silently in background

### Payment Method Validation

**Location-Based Restrictions**

```javascript
function validatePaymentMethods() {
  const city = selectedCity.toLowerCase();
  const isBogota = city.includes('bogot');
  
  if (!isBogota) {
    // Disable Cash on Delivery
    codInput.disabled = true;
    codContainer.classList.add('opacity-50', 'grayscale');
    showWarning("COD only available in Bogotá");
  }
}
```

Payment options dynamically enable/disable based on delivery city.

### Payment Processing Flow

<Tabs>
  <Tab title="MercadoPago">
    **Online Gateway Flow**
    
    1. Customer clicks "Pay with MercadoPago"
    2. System validates all form data
    3. Auto-saves address if first-time customer
    4. Calls Firebase Function `createMercadoPagoPreference`
    5. Function creates payment session in MercadoPago
    6. Customer redirected to MercadoPago checkout
    7. Customer completes payment
    8. MercadoPago webhook notifies backend
    9. Backend creates order in Firebase
    10. Customer redirected to success page
  </Tab>
  <Tab title="Cash on Delivery">
    **COD Flow**
    
    1. Customer clicks "Confirm COD Order"
    2. System validates Bogotá address
    3. Auto-saves address and profile updates
    4. Calls Firebase Function `createCODOrder`
    5. Function creates order with PENDIENTE_PAGO status
    6. Order appears in admin panel for approval
    7. Customer redirected to success page
    8. Admin processes order manually
  </Tab>
  <Tab title="ADDI">
    **ADDI Credit Flow**
    
    1. Customer enters ID and phone (required)
    2. Clicks "Pay with ADDI"
    3. System validates customer data
    4. Auto-saves profile and address
    5. Calls Firebase Function `createAddiCheckout`
    6. Function creates ADDI session
    7. Customer redirected to ADDI platform
    8. ADDI performs credit check
    9. Customer selects payment plan
    10. ADDI webhook notifies backend
    11. Backend creates order
    12. Customer redirected to success page
  </Tab>
  <Tab title="Sistecrédito">
    **Sistecrédito Flow**
    
    1. Customer enters ID and phone (required)
    2. Clicks "Pay with Sistecrédito"
    3. System validates customer data
    4. Auto-saves profile and address
    5. Calls Firebase Function `createSistecreditoCheckout`
    6. Function creates credit session
    7. Customer redirected to Sistecrédito portal
    8. Completes credit application
    9. Webhook notifies backend on approval
    10. Backend creates order
    11. Customer redirected to success page
  </Tab>
</Tabs>

## Smart Profile Auto-Save

### Automatic Profile Completion

The checkout process intelligently updates customer profiles:

**First-Time Buyers**

```javascript
if (userAddresses.length === 0) {
  // Automatically save address as default
  shouldSaveAddress = true;
  isFirstAddress = true;
  
  await updateDoc(doc(db, "users", userId), {
    addresses: arrayUnion(newAddress),
    name: enteredName,
    phone: enteredPhone,
    document: enteredDocument
  });
}
```

**Profile Gap Filling**

If customer profile is missing data (name, phone, document), checkout automatically fills it:

```javascript
let updates = {};
if (!userProfile.name) updates.name = checkoutFormName;
if (!userProfile.phone) updates.phone = checkoutFormPhone;
if (!userProfile.document) updates.document = checkoutFormDocument;

if (Object.keys(updates).length > 0) {
  await updateDoc(userRef, updates);
}
```

<Info>
This ensures customer profiles are always complete without requiring separate profile setup steps.
</Info>
